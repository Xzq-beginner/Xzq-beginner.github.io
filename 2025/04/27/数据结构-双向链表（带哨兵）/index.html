数据结构-双向链表（带哨兵）

双向带哨兵链表实现

相较于单向链表，双向链表提供了从两端遍历的能力，在场景适应性、操作效率等方面有明显优势。这里我将分享一个带有哨兵节点的 Java 双向链表实现。

一、核心架构

整个双向链表的设计围绕两个核心哨兵节点展开：头哨兵head和尾哨兵tail。它们具有以下特点：

  • 头哨兵head的next指针初始化时指向尾哨兵tail
  • 尾哨兵tail的prev指针初始化时指向头哨兵head
  • 哨兵节点不含实际数据（值为10000仅作标识），主要作用是简化边界条件处理

这种设计消除了单独处理头尾边界情况的复杂性，使插入、删除等操作逻辑更加统一。

二、节点类实现

节点类包含三个关键部分：

  1. prev指针，指向当前节点的前驱节点
  2. value存储节点的值
  3. next指针，指向当前节点的后继节点

三、构造方法

构造方法初始化双向链表，创建头尾哨兵节点并建立它们之间的连接关系，为后续操作奠定基础。

四、异常处理方法

提供统一的非法索引异常处理方法，简化代码结构，使异常信息更加明确。

五、核心操作实现

（一）根据索引查找节点

从头哨兵开始遍历链表，直到找到指定索引位置的节点或遍历到尾哨兵结束。这种实现保证了索引查找的线性时间复杂度。

（二）根据索引插入节点

插入操作通过以下步骤完成：

  1. 找到目标索引位置的前驱节点
  2. 创建新节点，并正确设置其prev和next指针
  3. 调整前驱节点和后继节点的指针，完成插入

（三）向头部添加节点

利用insert方法在链表头部添加节点，逻辑简洁明了。

（四）按索引删除元素

删除操作的核心逻辑是：

  1. 找到目标节点的前驱节点
  2. 调整前驱节点和目标节点后继节点的指针，跳过目标节点
  3. 通过这种方式从链表中移除目标节点

（五）删除第一个节点

利用remove方法删除链表头部节点，代码简洁高效。

（六）添加最后一个节点

在链表尾部添加节点，直接操作尾哨兵的prev指针，提高操作效率。

（七）删除最后一个节点

删除最后一个节点，只需调整尾哨兵和新最后一个节点的指针关系。

（八）根据索引获取元素值

通过findNode方法找到目标节点并返回其值，逻辑清晰简洁。

六、遍历方式

提供了三种遍历方式：

（一）基于while循环遍历

从头哨兵的下一个节点开始，直到遍历到尾哨兵结束。

（二）基于for循环遍历

与while循环遍历逻辑相同，只是语法结构有所差异。

（三）迭代器实现遍历

通过实现Iterable接口，使链表支持for-each遍历，提高了代码的可读性和易用性。

总结

​ 这个双向带哨兵链表的实现涵盖了链表的基本操作，如添加、删除、查找、遍历等。通过内部类 Node 来构建链表节点，使得链表的结构清晰、操作方便。希望这篇博客能帮助你更好地理解和使用双向带哨兵链表。
