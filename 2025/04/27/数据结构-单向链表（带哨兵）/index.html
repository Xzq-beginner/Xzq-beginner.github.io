数据结构-单向链表（带哨兵）

单向带哨兵链表的实现

​ 在数据结构中，链表是一种常见的线性表结构，它通过节点之间的指针链接来存储数据。今天，我将分享一个单向带哨兵链表的 Java 实现，并详细介绍其内部类以及各个功能方法。

内部类 Node：链表的基石

​ 在单向链表中，节点是构成链表的基本单元。我定义了一个内部类 Node，它代表链表中的一个节点。

  • 字段说明 ：value 用于存储该节点的值，next 是一个指向下一个节点的指针。
  • 构造方法 ：通过有参构造方法，我们可以在创建节点时直接指定其值和下一个节点。

这个 Node 类被声明为 private static，因为它只在 SinglyLinkedList 类内部使用，并且与 SinglyLinkedList 类本身无关。

哨兵节点：简化链表操作

​ 创建了一个哨兵节点作为链表的头部。

​ 哨兵节点是一个特殊节点，它不存储实际的数据，其主要作用是简化链表操作。例如，在插入、删除等操作中，不需要对头节点进行特殊处理，因为哨兵节点始终存在。

链表的基本操作

添加元素

  • 添加第一个元素 ：通过 addFirst 方法，我们可以将元素添加到链表的第一个位置。

​ 它实际上是调用了 insert 方法来完成添加操作。

  • 添加到最后一个元素 ：使用 addLast 方法将元素添加到链表的末尾。

​ 首先通过 findLast 方法找到当前链表的最后一个节点，然后将新节点添加到其后面。

插入元素

​ 通过 insert 方法可以在指定索引位置插入元素。

​ 它会先找到目标索引位置的上一个节点，然后创建新节点并调整指针来完成插入。

删除元素

  • 删除第一个元素 ：调用 removeFirst 方法可以删除链表的第一个元素。

​ 同样是借助 remove 方法来实现。

  • 根据索引删除元素 ：remove 方法可以根据指定索引删除对应的元素。

​ 找到目标索引位置的上一个节点后，调整指针跳过要删除的节点。

获取元素

​ 通过 get 方法可以根据索引获取链表中对应位置的元素值。

​ 它先找到对应索引的节点，然后返回该节点的值。

遍历链表

  • while 循环遍历 ：lope1 方法使用 while 循环来遍历链表。

​ 从哨兵节点的下一个节点开始，依次访问每个节点的值。

  • for 循环遍历 ：lope2 方法使用 for 循环来遍历链表。

​ 与 while 循环类似，也是从哨兵节点的下一个节点开始遍历。

  • 迭代器遍历 ：通过重写 iterator 方法，我们为链表提供了迭代器支持。

​ 使用迭代器可以更方便、安全地遍历链表。

辅助方法

  • 查找最后一个节点 ：findLast 方法用于找到链表的最后一个节点。

​ 从哨兵节点的下一个节点开始，一直找到最后一个节点。

  • 根据索引查找节点 ：findNode 方法可以根据索引查找链表中的节点。

​ 从哨兵节点开始，依次查找直到找到对应索引的节点。

  • 索引检查 ：illegalIndex 方法用于检查索引是否合法。

​ 当索引不合法时，抛出异常并给出提示信息。

总结

​ 这个单向带哨兵链表的实现涵盖了链表的基本操作，如添加、删除、查找、遍历等。通过内部类 Node 来构建链表节点，使得链表的结构清晰、操作方便。希望这篇博客能帮助你更好地理解和使用单向带哨兵链表。
